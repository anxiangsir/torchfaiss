TorchFAISS

Pure PyTorch Distributed KMeans with a FAISS-Compatible API.

FAISS Compatible Multi-GPU Pure PyTorch BF16 Ready GitHub

Overview

TorchFAISS is a high-performance, distributed clustering library designed to bridge the gap between PyTorch's native ecosystem and the industry-standard FAISS API. It allows researchers and practitioners to seamlessly drop in GPU-accelerated KMeans clustering without converting tensors to numpy arrays or leaving the PyTorch memory domain.

Why another KMeans?

While FAISS is incredibly fast, moving data back and forth between PyTorch CUDA tensors and FAISS GPU resources introduces severe pipeline bottlenecks. TorchFAISS keeps your tensors on the device, utilizes native PyTorch autograd-free operations, and natively supports DistributedDataParallel (DDP) for multi-node clustering.

Installation

Requires Python 3.8+ and PyTorch 2.0+.

bash 
pip install git+https://github.com/anxiangsir/torchfaiss.git

Quick Start

Single GPU Training

The API is designed to be a 1:1 drop-in replacement for faiss.Kmeans.

python 
import torch
from torchfaiss import TorchKmeans

# Generate dummy data directly on GPU
d, k = 128, 1024
x = torch.randn(100000, d, device='cuda')

# Initialize and train
kmeans = TorchKmeans(d=d, k=k, gpu=True, niter=20, verbose=True)
kmeans.train(x)

# Assign new points to clusters
D, I = kmeans.assign(x[:10])
print(f"Centroids shape: {kmeans.centroids.shape}")

Distributed (Multi-GPU/Multi-Node)

TorchFAISS leverages torch.distributed to synchronize centroids across devices. Simply pass local shards to train().

python 
import torch.distributed as dist
from torchfaiss import TorchKmeans

dist.init_process_group("nccl")
local_x = get_local_shard_from_loader() # [N_local, d]

kmeans = TorchKmeans(
    d=d, k=k, 
    gpu=True, 
    distributed=True # Enable all-reduce during training
)
# Each rank trains on its local shard; centroids stay synced
kmeans.train(local_x)

BF16 Acceleration

For Ampere and newer GPUs, BFloat16 matrix multiplications significantly speed up distance calculations.

python 
kmeans = TorchKmeans(d=d, k=k, gpu=True, bf16=True)

API Reference

class TorchKmeans( d, k, niter=20, nredo=1, ... )

The core clustering engine, mirroring faiss.Kmeans initialization. Handles device placement, distribution flags, and precision configurations.

Parameters

Parameter Type Default Description
d int required Dimension of the input vectors.
k int required Number of clusters (centroids).
niter int 20 Lloyd iterations per restart.
nredo int 1 Independent restarts (keeps the one with lowest objective).
verbose bool False Print training progress and objective value.
spherical bool False If True, L2-normalize centroids (cosine similarity clustering).
seed int 1234 Random seed for initial centroid selection.
max_points_per_centroid int 256 Maximum training points per centroid (handles subsampling).
min_points_per_centroid int 39 Minimum training points per centroid (emits warning if breached).
frozen_centroids bool False Skip centroid update step, only performs assignments.
gpu bool | int False True for auto GPU, int for specific device ID, False for CPU.
distributed bool False Use torch.distributed (requires initialized process group).
bf16 bool False Enable BFloat16 for distance matmul hot path (requires CUDA BF16 support).
def train( x: np.ndarray | torch.Tensor ) → None

Runs the KMeans algorithm on the input vectors.

  • x: Input data of shape [N, d]. If distributed mode is active, x should be the local shard for this rank.
  • Behavior: Automatically handles subsampling if N > k * max_points_per_centroid. Initializes centroids and runs niter Lloyd iterations.
def assign( x: np.ndarray | torch.Tensor, batch_size: int = 65536 ) → tuple[np.ndarray, np.ndarray]

Finds the nearest centroid for each vector in x.

  • x: Query vectors of shape [N, d].
  • batch_size: Chunk size for streaming batched processing to avoid out-of-memory errors on massive query sets.
Returns: (D, I) where D[i] is the squared L2 distance (or cosine dist) to the nearest centroid, and I[i] is the index of that centroid. Return type matches input type (Tensor if input is Tensor, else ndarray).
property centroids: np.ndarray | None

Access the learned centroids. Shape is [k, d]. Set to None before train() is called. To get them as a PyTorch tensor, access the internal state or convert via torch.from_numpy().

Benchmark Methodology

Both methods run on 8 GPUs with identical data (ImageNet CLIP features, d=768), k=1000, niter=20, seed=1234. FAISS uses faiss.Kmeans(gpu=8); TorchFAISS uses torchrun --nproc_per_node=8 with data-parallel Lloyd iterations. TorchFAISS shards data across GPUs; FAISS replicates the index to all GPUs. These are architecturally different parallelism strategies.

Design Philosophy

Machine learning engineering is shifting toward end-to-end PyTorch workflows. Tools like xformers and flash-attention demonstrate the power of native CUDA/PyTorch integration. TorchFAISS applies this philosophy to clustering:

  • No C++ extensions required: Uses highly optimized standard PyTorch ops (e.g. torch.cdist, baddbmm).
  • Distributed natively: Works out of the box with PyTorch's NCCL backend without complex MPI setups.
  • Zero-copy design: Avoids the costly tensor.cpu().numpy() and subsequent FAISS GPU reallocation cycles.
